# 故障排除指南

## 常见问题及解决方案

### 1. HTTP 401 认证失败错误

**错误信息**：`HTTP error! status: 401` 或 `认证失败 (401)`

**原因**：
- API Key无效或已过期
- API Key格式错误（包含多余空格或特殊字符）
- 账户余额不足
- API Key权限被限制
- 使用了错误的API端点

**解决方案**：
1. **检查API Key格式**：
   - 确保API Key不包含多余的空格
   - 检查是否完整复制了所有字符
   - 确保没有包含换行符或其他特殊字符

2. **重新获取API Key**：
   - **Novita AI**: [访问Novita AI官网](https://novita.ai/) → 注册/登录 → API管理 → 创建新的API Key
   - **OpenAI**: [访问OpenAI平台](https://platform.openai.com/api-keys) → 登录 → API Keys → Create new secret key
   - **Anthropic**: [访问Anthropic控制台](https://console.anthropic.com/) → 登录 → API Keys → Create Key
   - **Google**: [访问Google AI Studio](https://makersuite.google.com/app/apikey) → 登录 → Get API key

3. **检查账户状态**：
   - 确认账户余额充足
   - 检查API使用权限
   - 确认账户没有被限制

4. **测试API连接**：
   - 在插件设置中点击"测试API连接"
   - 查看详细的错误信息
   - 根据提示进行相应修复

### 2. 模型不支持错误

**错误信息**：`请求参数错误 (400)` 或 `模型不存在`

**原因**：
- 选择了错误的模型名称
- API提供商不支持该模型
- 模型名称拼写错误

**解决方案**：
1. **检查模型名称**：
   - 确保模型名称完全正确
   - 注意大小写和特殊字符
   - 参考官方文档确认模型名称

2. **选择合适的模型**：
   - **Novita AI**: minimaxai/minimax-m1-80k, minimaxai/minimax-m1-32k, minimaxai/minimax-m1-8k
   - **OpenAI**: gpt-4, gpt-4-turbo, gpt-3.5-turbo
   - **Anthropic**: claude-3-opus, claude-3-sonnet, claude-3-haiku
   - **Google**: gemini-pro, gemini-pro-vision

3. **检查API提供商**：
   - 确保选择了正确的API提供商
   - 确认API Key对应正确的服务

### 3. Extension context invalidated 错误

**错误信息**：`网络错误: Extension context invalidated`

**原因**：
- 扩展被重新加载或更新
- 浏览器重启后扩展上下文失效
- 扩展文件被修改后自动重新加载

**解决方案**：
1. **刷新页面**：按 F5 或 Ctrl+R 刷新当前页面
2. **重新加载扩展**：
   - 打开 `chrome://extensions/`
   - 找到"搜题答案助手"
   - 点击刷新图标
3. **重启浏览器**：完全关闭Chrome后重新打开

### 4. 插件按钮不显示

**问题**：页面右上角没有看到"识别题目"按钮

**解决方案**：
1. 刷新页面，等待1-2秒
2. 检查插件是否已启用
3. 检查控制台是否有错误信息
4. 重新加载扩展

### 5. API调用失败

**错误信息**：`API连接失败` 或 `网络错误`

**解决方案**：
1. 检查API Key是否正确
2. 确认网络连接正常
3. 验证API服务状态
4. 检查API调用频率限制
5. 确认选择了正确的API提供商和模型

### 6. 题目识别不准确

**问题**：插件无法正确识别题目

**解决方案**：
1. 手动选择题目文本
2. 确保题目文本长度在10-1000字符之间
3. 检查题目是否包含关键词
4. 尝试不同的题目格式

### 7. 答案显示框无法关闭

**问题**：点击关闭按钮没有反应

**解决方案**：
1. 按ESC键关闭
2. 点击答案框外的任意区域
3. 刷新页面重新加载插件

### 8. 自定义API配置问题

**问题**：自定义API无法正常工作

**解决方案**：
1. **检查API端点**：
   - 确保端点URL正确
   - 确认端点支持POST请求
   - 检查端点是否需要特殊认证

2. **检查请求格式**：
   - 确认API接受JSON格式
   - 检查请求体结构是否正确
   - 验证模型名称是否有效

3. **测试连接**：
   - 使用"测试API连接"功能
   - 查看详细的错误信息
   - 根据错误提示调整配置

## 调试步骤

### 步骤1：检查扩展状态
1. 打开 `chrome://extensions/`
2. 确认"搜题答案助手"已启用
3. 检查是否有错误信息

### 步骤2：检查控制台错误
1. 按F12打开开发者工具
2. 查看Console标签页
3. 寻找与插件相关的错误信息

### 步骤3：测试基本功能
1. 点击插件图标，检查弹出窗口
2. 在页面上选择文本，检查浮动按钮
3. 点击右上角识别按钮，检查功能

### 步骤4：验证API配置
1. 确认API提供商选择正确
2. 验证API Key格式和有效性
3. 检查模型名称是否正确
4. 测试API连接

### 步骤5：重新加载扩展
1. 在扩展管理页面点击刷新图标
2. 刷新当前网页
3. 重新测试功能

## 预防措施

### 1. 避免频繁修改扩展文件
- 修改代码后使用扩展管理页面的刷新功能
- 避免在开发过程中频繁重启浏览器

### 2. 保持稳定的网络连接
- 确保网络连接稳定
- 避免在网络不稳定时使用API功能

### 3. 定期检查API状态
- 定期验证API Key有效性
- 监控API调用频率
- 检查账户余额和权限

### 4. 合理选择模型
- 根据题目复杂度选择合适的模型
- 考虑成本和性能的平衡
- 定期测试不同模型的效果
